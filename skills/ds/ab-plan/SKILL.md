name: ab-plan
description: A/B test plan: hypothesis, metrics, power, SRM, ramp, logging, analysis.
---

When to use:
- You need a complete A/B test plan.

Inputs:
- Product goal, hypothesis, target metric, eligibility rules.

Output:
- Objective + directional hypothesis
- Primary metric + 2 to 3 guardrails
- Randomization unit + eligibility + exposure definition
- Power plan (MDE, alpha, power) with assumptions
- Runtime + ramp strategy
- SRM checks + instrumentation/logging checklist
- Analysis plan (estimator, covariates, segmentation, multiple testing notes)
- Decision rule + rollback criteria

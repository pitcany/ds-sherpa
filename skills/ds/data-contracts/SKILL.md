
---
name: data-contracts
description: Define data contracts for key datasets (schema, SLAs, owners).
---

When to use:
- You need a data contract for a key dataset.

Inputs:
- Dataset name, owner, SLA requirements, schema.

Output:
- Owner, SLA, refresh cadence
- Schema with types + required fields
- Invariants and unit tests
- Downstream dependencies
- Change management plan
- Return a contract spec ready for docs
